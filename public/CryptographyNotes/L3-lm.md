## 區塊加密法

### 主要概念

- **定義：** 將明文分割成固定大小的區塊，然後對每個區塊進行加密以產生相同大小的密文區塊。
- **區塊大小：** 常見的區塊大小為64或128位元。
- **與串流加密法的比較：** 區塊加密一次處理一個區塊，而串流加密一次處理一個位元或位元組。
- **應用：** 區塊加密廣泛應用於網路相關的對稱式密碼應用。
- **可逆性：** 為了使加密程序可逆（即可解密），每個明文區塊必須產生唯一的密文區塊。

### 理想的區塊加密法與夏農的貢獻

- **替代運算 (Substitution)：** 將資料的一部分代入回合函數，並與另一部分進行XOR運算，受到子金鑰影響。
- **重排運算 (Permutation)：** 在替代運算後，對資料的左右兩部分進行對調。
- **替代-重排網路：** 夏農提出的概念，是現代許多區塊加密法的基礎。
- **混淆 (Confusion)：** 使密文的統計結果與金鑰之間的關係盡可能複雜，以抵抗統計分析。
- **擴散 (Diffusion)：** 使明文的統計結構消失在密文的廣泛統計資訊裡，消除明文的統計特性。
- **乘積加密法：** 連續執行兩個或以上的基本加密法，以提高安全性。

### 菲仕托 (Feistel) 加密架構

- **設計者：** Horst Feistel。
- **基礎：** 可逆的乘積加密法。
- **輸入：** 長度為2w位元的明文區塊和金鑰K。
- **處理方式：** 將明文分成L0和R0兩部分，進行多個回合的迭代處理。
- **每回合操作：** 左半部 (Li) 等於前一回合的右半部 (Ri-1)。
- 右半部 (Ri) 是前一回合的左半部 (Li-1) 與將前一回合的右半部 (Ri-1) 和子金鑰 (Ki) 輸入回合函數 F 後的結果進行 XOR 運算。
- 回合後交換左右兩半資料（除了最後一回合）。
- **回合函數 (Round Function) F：** 每回合都相同，但輸出受該回合的子金鑰影響。
- **實作夏農的替代與重排：** 替代透過回合函數F實現，重排透過每回合結束時的左右半部交換實現。

### Feistel 加密法的設計原理

- **區段大小：** 增加區段大小通常能提高安全性，但會減慢加解密速度（傳統為64位元）。
- **金鑰大小：** 增加金鑰大小能提高安全性，使暴力破解更困難，但也會減慢加解密速度。
- **回合數：** 增加回合數能提高安全性，但會減慢加解密速度。
- **子金鑰的產生：** 複雜的子金鑰產生方法能使分析更困難，但也可能減慢加解密速度。
- **回合函數：** 複雜的回合函數能使分析更困難，但也可能減慢加解密速度。
- **軟體實現與分析：** 設計時需考慮便於軟體快速加解密以及易於分析其安全性。

### Feistel 加密法的基本原理與加解密動作

- **加密動作：** Li+1 = Ri
- Ri+1 = Li ⊕ F(Ki+1，Ri)
- **解密動作：** Li = Ri+1 ⊕ F(Ki+1，Li+1)
- Ri = Li+1
- **子金鑰：** 由秘密金鑰產生，並在每一回合中使用不同的子金鑰。解密時子金鑰的使用順序與加密時相反。

## 資料加密標準（DES）

### 概述

- **廣泛使用：** 目前最廣為使用的區塊加密法之一。
- **採用時間：** 1976年由美國國家標準局（NBS，現為NIST）採用。
- **標準：** NIST發布的第46項聯邦資訊處理標準（FIPS PUB 46）。
- **金鑰與區塊大小：** 使用56位元的金鑰加密64位元的資料區塊。
- **爭議：** 因其廣泛使用和金鑰長度等問題，安全性一直是爭論的焦點。

### DES的歷史

- **Lucifer加密法：** IBM在1971年開發，由Feistel在60年代末期主持的計畫，使用128位元金鑰和64位元資料區塊。
- **與NSA的合作：** IBM與美國國家安全局（NSA）的專家合作開發商業化產品，目標是實現在單晶片中（最初希望是64位元金鑰）。
- **NBS公開徵求：** 美國國家標準局在1973年公開徵求美國國家加密標準。
- **IBM的參與：** IBM以Tuchman和Meyer的計畫成果（改版的Lucifer）參與徵選，並在1976年被採納為DES。

### DES的設計爭議

- **金鑰長度縮減：** Lucifer使用128位元金鑰，但提交的版本卻是56位元金鑰，引起安全疑慮。
- **S-盒子的機密性：** DES內部架構的設計規範（S-boxes）被列為機密，使用者無法確認是否存在NSA可能利用的弱點。
- **內部架構的強度：** 後來的研究顯示DES的內部架構非常強固。
- **廣泛使用：** 業界，特別是金融方面，仍然廣泛使用DES。
- **NIST的建議：** NIST建議DES僅用於傳統舊系統，新系統應改用3DES。

### DES加密系統

- **處理大於64位元的資料：** 將明文或密文每64位元視為一個區塊進行切割，然後對每個區塊進行加密或解密。
- **最後一個區塊的處理：** 如果最後一個區塊大小小於64位元，需要附加“0”位元使其達到64位元。
- **金鑰大小：** DES所用的加密或解密金鑰是64位元大小，但其中有8個位元用於錯誤更正，實際有效的金鑰長度只有56位元。

### DES加密流程

1. **輸入明文：** 64位元。
2. **啟動重排（IP）：** 對輸入的64位元進行初始排列。
3. **進行16回合功能運算：** 每回合使用不同的48位元子金鑰，並將資料切分成左右各32位元的區塊進行處理。
4. **交換32位元區塊：** 在最後一回合之前，將左右32位元的區塊互換。
5. **啟動重排的逆運算（IP-1）：** 執行初始排列的逆運算，得到最終的64位元密文。

### 啟始重排程序（IP）

- **第一階段：** DES加密運算的第一個步驟。
- **位元重組：** 重新排列輸入的64位元。
- **左右分割：** 將偶數位元移到左半邊（LH），奇數位元移到右半邊（RH），各32位元。
- **硬體實現：** 結構規律，易於用硬體實現。

### DES的回合結構

- **資料分割：** 使用L（左）、R（右）各32位元的資料。
- **回合方程式：** Li = Ri–1
- Ri = Li–1 ⊕ F(Ri–1, Ki)
- **回合金鑰：** 每回合使用一個48位元的子金鑰Ki。

1. **F函數運算：擴充式重排（E）：** 將32位元的R擴充為48位元。
2. **XOR運算：** 將擴充後的48位元R'與48位元的子金鑰Ki進行XOR運算。
3. **替代程序（S-boxes）：** 將XOR結果的48位元輸入8個S-boxes，每個S-box將6位元輸入轉換為4位元輸出，總共產生32位元的輸出。
4. **重排函數（P）：** 對S-boxes產生的32位元輸出進行固定的排列。

### DES - 擴充函數(E)

- **目的：** 將32位元的輸入擴充為48位元，同時進行位元的重排和重複。
- **表格定義：** 定義了輸入的哪些位元會被映射到輸出的哪些位元，有些輸入位元會被重複使用。

### 替代運算 S-boxes

- **組成：** 由8個S-boxes組成。
- **輸入與輸出：** 每個S-box接受6位元的輸入，產生4位元的輸出。
- **查表方式：** 輸入的第一個和最後一個位元組成2位元的列索引，中間4個位元組成4位元的行索引，根據行列索引在S-box表格中查找對應的4位元輸出。
- **非線性：** S-boxes的轉換是非線性的，這是DES安全性的重要來源。

### DES - F函數排列函數(P)

- **目的：** 對S-boxes產生的32位元輸出進行固定的排列。
- **表格定義：** 定義了輸入的每個位元會被映射到輸出的哪個位置。

### 產生金鑰

- **子金鑰產生：** 為每一回合產生一個48位元的子金鑰。
- **PC-1（重排選擇1）：** 將64位元的原始金鑰縮減並重新排列為56位元，同時丟棄每8個位元作為奇偶校驗位。
- **左右分割：** 將56位元的金鑰分成兩個28位元的部分（C0和D0）。
- **循環左移（LS）：** 在每一回合，C和D分別進行循環左移一或兩個位元，移動的位數根據回合數而定。
- **PC-2（重排選擇2）：** 將循環左移後的56位元（Ci和Di合併）縮減並重新排列為48位元的子金鑰Ki。

### DES 子鑰匙產生

- **主金鑰到子金鑰：** 從56位元的主金鑰產生16個48位元的子金鑰。

1. **步驟：** 使用PC-1將64位元主金鑰轉換為兩個28位元的區塊C0和D0。
2. 對於每個回合i（從1到16）：

- 根據該回合的左移位數表，對Ci-1和Di-1分別進行循環左移，得到Ci和Di。
- 將Ci和Di合併為一個56位元的區塊。
- 使用PC-2對合併後的56位元區塊進行縮減和排列，得到48位元的子金鑰Ki。

### DES的左移位數表

- 定義了在每一回合中，C和D需要循環左移的位元數，通常為1或2位元。

### DES回合金鑰的產生（範例說明）

- 通過一個選定的秘密金鑰（如ASCII編碼的"[science]"）的二進制表示，經過初始金鑰排列（PC-1）得到C0和D0，然後根據每回合的左移位數表進行循環左移，再經過壓縮排列（PC-2）得到每一回合的子金鑰K1, K2, ..., K16。

### DES 加密法- 例子

- 展示了如何使用DES對一個64位元的明文（如"[security]"的ASCII編碼）進行加密，包括初始排列（IP）、第一回合的F函數運算（擴充、與子金鑰XOR、S-盒替代、P盒排列）以及與左半部分進行XOR運算等步驟。

### DES 的解密方式

- **演算法相同：** 解密演算法與加密幾乎相同。
- **子金鑰反向使用：** 主要區別在於解密時子金鑰的使用順序與加密時相反，即從K16到K1。
- **Feistel結構：** 由於Feistel結構的特性，逆向使用子金鑰即可還原明文。
- **初始和終結排列：** 初始排列（IP）和終結排列（IP-1）在解密過程中仍然執行，以還原資料的原始順序。

## 四種加密模式

### 秘密金鑰密碼系統加密模式

- **ECB模式 (Electronic Code Book Mode)：** 電子密碼本模式。
- **CBC模式 (Cipher Block Chaining Mode)：** 密碼區塊鏈模式。
- **CFB模式 (Cipher Feedback Mode)：** 密碼回饋模式。
- **OFB模式 (Output Feedback Mode)：** 輸出回饋模式。

### ECB Mode (Electronic Code Book Mode)

- **加密：** 將明文分成若干區塊，每個區塊使用相同的金鑰獨立加密。
- 密文區塊i = 加密(明文區塊i)
- **優點：** 可以平行處理。
- **缺點：** 相同的明文區塊會產生相同的密文區塊，容易遭受替換攻擊和模式分析。
- **解密：** 對每個密文區塊使用相同的金鑰獨立解密。
- 明文區塊i = 解密(密文區塊i)

### CBC Mode (Cipher Block Chaining Mode)

- **加密：** 每個明文區塊在加密前都與前一個密文區塊進行XOR運算（第一個明文區塊與一個稱為初始化向量IV進行XOR運算）。然後對結果進行加密。
- 密文區塊1 = 加密(明文區塊1 ⊕ IV)
- 密文區塊i = 加密(明文區塊i ⊕ 密文區塊i-1)
- **優點：** 相同的明文區塊由於與前一個密文區塊（或IV）進行XOR，會產生不同的密文，避免了ECB模式的缺陷。
- **缺點：** 加密過程不能平行處理，因為每個區塊的加密都依賴於前一個區塊的密文；解密可以平行處理；如果密文中的一個位元錯誤，會影響到當前區塊和下一個區塊的解密。

### CFB Mode (Cipher Feedback Mode)

- **加密：** 前一個密文區塊被加密，然後其輸出與當前明文區塊進行XOR運算以產生當前密文區塊（第一個明文區塊與加密後的IV進行XOR運算）。
- 密文區塊1 = 明文區塊1 ⊕ 加密(IV)
- 密文區塊i = 加密(密文區塊i-1) ⊕ 明文區塊i
- **優點：** 一個密文區塊的錯誤只會影響當前區塊和下一個區塊的解密；可以設計出區塊大小小於加密演算法原生區塊大小的固定加密法。
- **缺點：** 加密和解密都不能平行處理。

### OFB Mode (Output Feedback Mode)

- **加密：** 一個稱為初始化向量IV的值被加密，其輸出與第一個明文區塊進行XOR運算產生第一個密文區塊。然後，前一個加密的輸出作為下一次加密的輸入，其輸出再與下一個明文區塊進行XOR運算，依此類推。
- S0 = 加密(IV)
- Si = 加密(Si-1)
- 密文區塊i = Si ⊕ 明文區塊i
- **解密：** 過程與加密類似，只是將密文與相同的金鑰流（Si）進行XOR運算以恢復明文。
- S0 = 加密(IV)
- Si = 加密(Si-1)
- 明文區塊i = Si ⊕ 密文區塊i
- **優點：** 每個密文區塊的產生都是獨立的，不受前面明文區塊的影響。
- **缺點：** 如果IV重複使用，可能會危及安全性。

## 崩塌效應 (Avalanche Effect)

- **關鍵特性：** 加密演算法所需具備的重要特性。
- **定義：** 輸入（明文或金鑰）中即使只有一個位元的微小改變，也會導致輸出（密文）中約一半的位元發生改變。
- **重要性：** 強大的崩塌效應使得對密文的統計分析難以反推明文或金鑰的微小變化，有助於抵抗差分密碼分析等攻擊。
- **DES的崩塌效應：** DES被認為具有很強的崩塌效應。

## 三重DES密碼系統（Triple DES）

### Triple DES 加密架構

- **基本原理：** 使用DES演算法執行三次加密操作，通常涉及兩個或三個不同的金鑰（K1、K2、K3）。
- **常見形式：3DES (EEE)：** 加密-加密-加密，使用三個不同的金鑰：密文 = E(K3, E(K2, E(K1, 明文)))。
- **3DES (EDE)：** 加密-解密-加密，可以使用兩個或三個不同的金鑰：密文 = E(K3, D(K2, E(K1, 明文)))。EDE形式更常用，且當K1=K2=K3時，可以兼容標準的DES。

### Triple DES

- **與DES的兼容性：** 3DES的一個優點是舊的DES用戶可以在3DES系統中使用，只需設置K1=K2=K3=K，此時3DES等同於單次DES加密。
- **金鑰長度：** 使用3個不同的金鑰時，有效金鑰長度為168位元。
- 允許只用兩把金鑰（K1=K3），此時有效金鑰長度為112位元。
- **FIPS批准：** 美國聯邦資訊處理標準（FIPS）批准了3DES作為對稱編碼（加密）演算法。
- **DES的替代：** 之前遺留的DES只能使用在舊系統，新採購的系統必須支持3DES。
- **向AES過渡：** DES和AES將共存作為FIPS批准的演算法，但未來將逐漸轉向使用AES。

### 3DES的優/缺點

- **3DES的優點：** 由於使用與DES相同的演算法，因此對密碼分析具有與DES相似的抵抗力。
- 比DES更長的金鑰長度（168位元或112位元）可以有效抵抗暴力破解法。
- **3DES的缺點：** 速度較慢：由於需要執行三次DES演算法，其速度比DES慢三倍，不適用於對效能要求很高的應用。
- 區塊大小較小：仍然使用64位元的區塊大小，可能在某些應用中不夠靈活。

### 區段加密法的設計原理

- **基本法則：** 仍然遵循1970年代Feistel提出的基本架構。
- **回合個數：** 回合數越多越好，以抵抗暴力破解等攻擊。
- **函數 f：** 應提供良好的“混淆”特性、非線性以及強大的崩塌效應。
- **產生金鑰演算法：** 應設計複雜的子金鑰產生方式，並確保金鑰具有良好的崩塌效應。
- **運算模式：** 區塊加密法針對固定大小的區段進行加密，對於任意長度的資訊，需要使用不同的運算模式（如ECB、CBC、CFB、OFB）。

### 菲仕托加密法的強度

- **來源：** 強度來自回合數、函數F的設計以及產生金鑰的演算法。
- **回合數的重要性：** 即使函數F的設計相對較弱，只要回合數足夠多，也能提供較高的安全性。
- **函數F的基本原則：** 函數F的演算法應具有很好的崩塌效應，且其結果應難以還原。
- **S-盒子的特性：** S-boxes之間的關係應該是非線性的，且無法以線性函數近似模擬。
- **金鑰產生的安全性：** 金鑰的產生方式應確保金鑰與密文之間滿足完全崩塌準則和位元獨立準則。

### 暴力破解

- **方法：** 嘗試所有可能的金鑰，直到找到正確的金鑰能夠解密密文得到有意義的明文。
- **平均嘗試次數：** 平均需要嘗試的金鑰數量大約是可能的金鑰總數的一半。
- **金鑰長度與破解時間的關係：** 表格展示了不同金鑰長度下，暴力破解所需的時間隨著金鑰長度的增加呈指數級增長。較長的金鑰長度能有效抵抗暴力破解。

### DES的強度 － 金鑰長度

- **可能的金鑰總數：** 56位元的金鑰有2^56（約7.2 × 10^16）種可能的組合。
- **早期的不可行性：** 表面上看來，如此巨大的金鑰空間使得暴力攻擊在早期似乎不可行。
- **近期進展：** 然而，近年來的技術進步表明暴力破解DES在現實中是可能的。
- **實際破解時間：** 在1997年，破解DES只需要幾個月；1998年使用特定硬體（如EFF的Deep Crack）只需要幾天；1999年也只需要幾天的時間。
- **破解的挑戰：** 破解金鑰不僅是找出所有可能的金鑰，還必須能夠正確辨認出解密後的明文是否正確。
- **替代方案的必要性：** 由於DES金鑰長度過短，已經必須考慮使用更安全的替代方案，如3DES或AES。

### DES演算法的本質

- **破解的考量：** 需要考慮攻擊者根據DES演算法的特性而可能進行的破解。
- **S-盒子的重要性：** 重點在每個回合重複使用的8個取代表格（S-boxes）的設計特性，其非線性是抵抗線性密碼分析的關鍵。

### DES 的強度 – 分析破解法

- **存在的攻擊方法：** 有許多針對DES的分析破解方式，如差分密碼分析和線性密碼分析。
- **原理：** 這些破解方法依賴於對DES細部的加密結構進行分析，通過蒐集與加密法相關的資訊（如明文-密文對），最終目標是解出部分或全部的子金鑰位元。
- **與暴力破解結合：** 如果分析破解能夠找到部分子金鑰位元，那麼可以使用窮舉法搜索剩餘的位元，從而降低暴力破解的複雜度。
- **統計攻擊法：** 這些方法在本質上都屬於統計攻擊法，利用輸入和輸出之間的統計規律來推導金鑰資訊。

## 串流加密

### 概述

- **加密方式：** 一次加密明文的一個位元組（或一個位元）。
- **金鑰串流：** 架構通常依賴於一個以秘密金鑰為輸入的虛擬亂數位元產生器，產生一連串看似隨機的位元或位元組序列，稱為「金鑰串流」。
- **XOR運算：** 明文與金鑰串流進行XOR運算以產生密文：Ci = Mi XOR StreamKeyi。
- **與一次性密碼本的比較：** 串流加密法類似於一次性密碼本加密法，但不同的是一次性密碼本使用真正的亂數串流，而串流加密法使用虛擬亂數串流。
- **適用場景：** 串流加密法通常適用於網路連結或瀏覽器等需要高速且連續加密的場景。
- **區塊加密適用場景：** 區塊加密更適用於（檔案、電子郵件、資料庫）等需要處理固定大小資料塊的場景。

### 串流加密結構

- **核心：** 串流加密法的核心是虛擬亂數位元產生器（PRBG）。
- **目標：** 從一個短的金鑰生成一個次序夠亂且長度夠長的二元訊號（金鑰序列），目標是使金鑰序列難以預測。

### 串流加密法(Stream Ciphers)

- **虛擬亂數位元產生器（PRBG）：** 是串流加密的關鍵，用於從短金鑰產生看似隨機的長金鑰序列（keystream）。
- **安全需求：** 金鑰序列產生器通常需要滿足以下三項安全需求：

1. **金鑰序列週期要夠大：** 應超過訊息長度，以避免金鑰序列的重複使用。
2. **金鑰序列要容易計算產生：** 確保加密和解密的效率。
3. **金鑰序列難被預測：** 給定金鑰序列的一部分，在計算上應該很難預測後續的位元。理想情況下，給定前n位元輸出，第n+1位元是0或1的機率應各佔百分之50。

- **安全性評估：** 目前沒有一套世界標準與檢測方法可以完全確定PRBG是否難以被預測，通常使用各種已知的攻擊方式進行測試，目前還沒有通用的破解理論。

### 串流加密法的特性

- **設計考量：** 加密序列應該要有較長的週期，以避免重複使用。
- 金鑰串流應該儘可能接近真正的亂數串流，使其難以預測。
- **安全性與金鑰長度：** 虛擬亂數產生器的結果完全由輸入金鑰的值決定。如果金鑰長度足夠，且PRBG設計良好，串流加密法可以達到與區塊加密法相當的安全性。
- **主要優點：** 速度更快且程式碼更少，這使得它們在資源受限的環境中很有用。
- **效能比較：** 表格中列出了DES、3DES和RC4的典型金鑰長度和加密速度（Mbps），可以看出RC4作為一種串流加密法，通常具有較高的速度。

### RC4

- **設計者與時間：** 由Ron Rivest在1987年為RSA Security公司設計的串流加密法。
- **操作單位與金鑰長度：** 以位元組為運作單位，且金鑰長度是可變動的（通常介於40到256位元之間）。
- **公開：** 演算法的細節在匿名者公開後變得廣為人知。
- **基本原理：** 維護一個狀態陣列S（通常是256位元組），並通過一系列操作（包括交換S中的元素）來生成金鑰串流。金鑰串流的每個位元組都從S中選取出來，然後與明文的對應位元組進行XOR運算。每次產生一個金鑰串流位元組後，S的狀態都會被更新。

### RC4 金鑰排程 (Key Scheduling Algorithm, KSA)

- **初始化S和T：** 將狀態陣列S的元素初始化為從0到255的遞增序列（S[i] = i for i from 0 to 255）。
- 創建一個臨時陣列T。如果金鑰K的長度為256位元組，則將K複製到T；否則，對於長度為keylen位元組的金鑰K，將K的前keylen個元素複製到T的前keylen個元素，然後重複K直到填滿T（T[i] = K[i mod keylen]）。

1. **擾亂S：** 使用T來擾亂S的初始狀態。通過另一個循環遍歷S，並根據S和T的當前元素來交換S中的元素：
2. j = 0
3. for i = 0 to 255 do
4. j = (j + S[i] + T[i]) mod 256
5. swap(S[i], S[j])
6. 完成此步驟後，金鑰K和臨時陣列T就不再使用，S被用於生成金鑰串流。

### RC4 加密

- **金鑰串流生成：** 在金鑰排程完成後，通過以下步驟生成金鑰串流並進行加密：
- i = 0
- j = 0
- for each message byte Mi do
- i = (i + 1) mod 256
- j = (j + S[i]) mod 256
- swap(S[i], S[j])
- t = (S[i] + S[j]) mod 256
- Ci = Mi XOR S[t]
- 這個過程會不斷循環使用S的所有元素，並且S的狀態會根據其當前結構進行更新。每個產生的金鑰串流位元組S[t]都與明文位元組Mi進行XOR運算，得到密文位元組Ci。

## 區塊加密法測驗

### 短答題

1. 請簡述區塊加密法與串流加密法在處理明文上的主要差異。

5. 簡要解釋DES加密流程中的初始排列（IP）和逆初始排列（IP-1）的作用。
6. 在DES的回合結構中，擴充函數E和替代盒（S-boxes）各自的功能是什麼？

10. 為什麼三重DES（3DES）被認為比單個DES更安全？它有哪些主要的缺點？

### 簡答題答案

1. 區塊加密法將明文分割成固定大小的區塊，然後對每個區塊獨立進行加密，產生相同大小的密文區塊。而串流加密法則是將明文視為一個連續的資料流，一次加密一個位元或一個位元組，通常通過與一個由金鑰生成的金鑰串流進行XOR運算實現。
2. 菲仕托加密架構是一種對稱式加密結構，它將明文區塊分成左右兩半，並進行多輪迭代處理。在每一輪中，右半部分作為一個輸入傳遞給一個回合函數F，該函數的輸出與左半部分進行XOR運算，結果成為新的右半部分，而原來的右半部分則成為新的左半部分。
3. 在菲仕托加密架構中，回合函數F的主要作用是對輸入的資料（通常是右半部分和一個子金鑰）進行混淆和擴散，以增加密文的複雜性和安全性。F函數的輸出與左半部分進行XOR運算，使得每一輪的結果都依賴於前一輪的資料和當前的子金鑰。
4. 資料加密標準（DES）的區塊大小是64位元，金鑰大小名義上是64位元，但其中8位元用於奇偶校驗，因此有效的金鑰長度是56位元。
5. DES加密流程中的初始排列（IP）是對64位元的明文輸入進行一個固定的位元重排，將輸入重新排列成一個特定的順序，分為左右兩個32位元的區塊。逆初始排列（IP-1）則是在16輪加密運算完成後，對最終的64位元輸出執行IP的逆向操作，恢復原始的位元順序以產生最終的密文。
6. 在DES的回合結構中，擴充函數E將32位元的輸入區塊擴充為48位元，通過複製某些位元來增加長度，並進行重排，目的是使輸入可以與48位元的子金鑰進行XOR運算，並為後續的S-盒替代提供更大的輸入。替代盒（S-boxes）是DES中唯一的非線性組件，它們接收6位元的輸入並產生4位元的輸出，通過查表的方式實現替代，引入混淆性，是抵抗線性密碼分析的關鍵。
7. 在DES中，產生每一回合所需的子金鑰的過程如下：首先，64位元的主金鑰經過重排選擇1（PC-1）的處理，丟棄奇偶校驗位並重新排列為56位元。然後，這56位元被分成左右兩個28位元的區塊。在每一回合中，這兩個28位元的區塊分別進行循環左移（位移的位數根據回合數而定），然後經過重排選擇2（PC-2）的處理，縮減為48位元的子金鑰供該回合使用。
8. ECB（電子密碼本）模式對每個明文區塊使用相同的金鑰獨立加密，相同的明文區塊會產生相同的密文區塊，這使得它容易受到模式分析和替換攻擊；其優點是可以並行處理。CBC（密碼區塊鏈）模式在加密每個明文區塊之前，會先與前一個密文區塊進行XOR運算（第一個區塊與初始化向量IV進行XOR），這使得即使相同的明文區塊也會產生不同的密文區塊，提高了安全性，但加密過程不能並行處理，且密文的錯誤會傳播到後續的解密。
9. 加密演算法的崩塌效應是指當輸入（明文或金鑰）發生極其微小的改變（例如，僅改變一個位元）時，輸出（密文）會發生顯著且不可預測的變化，通常輸出中大約一半的位元會發生翻轉。這是一個重要的安全特性，因為它使得攻擊者難以通過分析明文和密文之間的細微差別來推導出金鑰或明文的其他部分，有助於抵抗差分密碼分析等攻擊。
10. 三重DES（3DES）被認為比單個DES更安全，主要是因為它使用了更長的有效金鑰長度（通常是112位元或168位元），從而大大增加了暴力破解的難度，有效抵抗了單個DES因金鑰長度過短而容易遭受暴力破解的風險。然而，3DES的主要缺點是加密速度相對較慢，因為它需要執行三次DES加密操作，這在對效能要求較高的應用中可能成為瓶頸，並且其仍然基於64位元的區塊大小。

## 術語表

- **區塊加密法 (Block Cipher)：** 一種對固定大小的明文區塊進行加密的對稱加密演算法。
- **串流加密法 (Stream Cipher)：** 一種一次加密明文的一個位元或位元組的對稱加密演算法，通常使用金鑰串流和XOR運算。
- **資料加密標準 (DES)：** 一種早期的廣泛使用的區塊加密標準，使用56位元金鑰和64位元區塊。
- **三重DES (3DES)：** 一種通過執行三次DES加密操作來增強安全性的區塊加密演算法。
- **菲仕托 (Feistel) 加密架構：** 一種將加密過程分解為多個回合的對稱加密結構，每回合使用一個回合函數和子金鑰。
- **明文 (Plaintext)：** 未經加密的原始訊息。
- **密文 (Ciphertext)：** 經過加密處理後的訊息。
- **金鑰 (Key)：** 在加密和解密過程中使用的秘密資訊。
- **加密 (Encryption)：** 將明文轉換為密文的過程。
- **解密 (Decryption)：** 將密文轉換回明文的過程。
- **回合函數 (Round Function)：** 在菲仕托加密架構的每一回合中執行的轉換函數。
- **子金鑰 (Subkey)：** 從主金鑰派生出來，在每一回合加密或解密中使用的金鑰。
- **替代 (Substitution)：** 將一個符號或位元組替換為另一個符號或位元組的操作。
- **重排 (Permutation)：** 重新排列資料中元素（如位元）的順序的操作。
- **混淆 (Confusion)：** 使密文的統計特性與金鑰之間的關係複雜化，以抵抗統計分析。
- **擴散 (Diffusion)：** 將明文的統計結構分散到密文中，使得明文的微小變化影響到密文的許多部分。
- **初始排列 (Initial Permutation, IP)：** 在DES加密開始時執行的固定位元重排。
- **逆初始排列 (Inverse Initial Permutation, IP-1)：** 在DES加密結束時執行的初始排列的逆操作。
- **S-盒 (Substitution Box)：** DES中用於執行非線性替代操作的組件。
- **P-盒 (Permutation Box)：** DES中用於執行固定位元重排操作的組件。
- **金鑰排程 (Key Schedule)：** 從主金鑰生成每一回合所需的子金鑰的過程。
- **加密模式 (Mode of Operation)：** 將區塊加密應用於大於區塊大小的資料或實現特定安全目標的方法，如ECB、CBC、CFB、OFB等。
- **初始化向量 (Initialization Vector, IV)：** 在某些加密模式中使用的隨機或偽隨機數，用於增加加密的隨機性。
- **崩塌效應 (Avalanche Effect)：** 加密演算法的一個特性，指輸入的微小變化導致輸出的顯著變化。
- **暴力破解 (Brute-force Attack)：** 嘗試所有可能的金鑰來解密密文的攻擊方法。
- **金鑰長度 (Key Length)：** 金鑰中位元的數量，通常決定了抵抗暴力破解的強度。
- **虛擬亂數產生器 (Pseudo-Random Bit Generator, PRBG)：** 一種使用確定性演算法從短的隨機種子生成看似隨機的位元序列的演算法。
- **金鑰串流 (Keystream)：** 由串流加密法中的PRBG生成的看似隨機的位元或位元組序列，用於與明文進行XOR運算。
- **RC4：** 一種廣泛使用的串流加密演算法。
