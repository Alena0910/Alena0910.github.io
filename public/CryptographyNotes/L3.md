## 區塊加密法
- 提供保密 / 認證
- 一定要**可逆** ( 不可逆為單向雜湊 )
- 將一小段明文視為一體，產生長度相同的密文區塊
- 區塊大小通常為 64 或 128 位元
- 明文區塊可能有2 ^ n 種，每個明文區塊產生唯一密文區塊才可逆

### 對稱式區塊加密演算法
以菲仕托 Feistel 區塊加密架構為基礎  

- 強度來自

## 資料加密標準 DES
- DES 的強度取決於金鑰長度

> ECB Mode  
> Electronic Code Book Mode  
> - 可以平行處理
> - 相同明文產生相同的密文，內容可被剪貼替換

> CBC Mode  
> Cipher Block Chaining Mode  

> CFB Mode  
> Cipher Feedback Mode  

> OFB Mode  
> Output Feedback Mode

### 崩塌效應 avalanche effect
- 改變輸入或 key 的一個位元，就會改變約一半輸出的位元

### 3DES

- 金鑰長度 168-bit (56-bit * 3)

優點
- 對密碼分析與 DES 有同樣的抵抗力
- 比DES金鑰長度更長

缺點
- 速度慢
- 區塊小(64-bit)

## 串流加密法

- 從一個短的金鑰產生一組次序夠亂且長度夠長的二元訊號
- 加密序列應要有較長周期
- 金鑰串流應該盡可能接近真正的亂數串流
- 速度更快
- 程式碼更少

### RC4
- 以位元組為運作單位
- 金鑰長度可變動
